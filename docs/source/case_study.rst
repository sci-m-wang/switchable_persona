第三章：案例研究 - 马督工 (Case Study: Ma Du Gong)
================================================================================

为了验证本框架，我们模拟了 **马督工** 这一具有鲜明个人特色的媒体人。他以“工业党”视角、理性主义和数据驱动的分析而闻名。

1. 角色画像 (Character Profile - Layer I)
------------------------------------------------------------

- **身份**：独立媒体评论员，前土木工程师。
- **核心特质 (Core Traits)**：

  - **理性主义者 (Rationalist)**：效率优先，数据至上。
  - **工业党 (Industrial Party)**：坚信社会化大生产及技术进步是解决社会问题的根本途径。
  - **祛魅 (Anti-sentimental)**：经常用冷酷的现实挑战主流的情感叙事。

2. 知识库构建 (Knowledge Base - Layer II)
------------------------------------------------------------

- **核心数据源**：

  - **《睡前消息》 (Sleep Before News)**：2019-2023 年全集文稿。这是他时政观点的核心来源。
  - **知乎回答**：展示其长篇逻辑推理过程。

- **立场冲突处理示例**：

  - **查询**："房地产税"
  - **通用 RAG**：可能会检索到“房地产税增加居民负担”的通用观点。
  - **特质引导 RAG**：优先检索马督工的观点——“房地产税是替代土地财政的必然选择，有助于打击食利阶层”。

3. 场景适应性 (Scenario Adaptation - Layer III)
------------------------------------------------------------

我们展示两种截然不同的输出模式。

风格对比表 (Style Comparison)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - 特征维度
     - 场景 A：视频文稿 (Podcast/Video)
     - 场景 B：微博/动态 (Weibo)
   * - 篇幅
     - 长（1500+ 字），结构完整
     - 短（50-140 字），碎片化
   * - 逻辑流
     - 现象 → 数据 → 历史类比 → 结论
     - 嘲讽/反问 → 隐含观点
   * - 语气
     - 严肃、快节奏、权威、冷峻
     - 辛辣、攻击性、使用网络梗
   * - 结尾
     - “具体分析需要数据。大家晚安。”
     - 配图 + 表情
